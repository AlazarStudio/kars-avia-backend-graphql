# Инструкция по развёртыванию KarsAvia

**Программное обеспечение:** KarsAvia — система автоматизации размещения экипажей авиакомпаний  
**Версия:** 3.4.0  
**Способ развёртывания:** Docker Compose (локальный запуск одной командой)

---

## 1. Требования к системе

| Компонент | Минимальная версия | Проверка версии |
|-----------|-------------------|-----------------|
| Docker Desktop (Windows/macOS) или Docker Engine (Linux) | 24.0+ | `docker --version` |
| Docker Compose | 2.20+ | `docker compose version` |
| Оперативная память | 4 ГБ свободных | — |
| Дисковое пространство | 5 ГБ | — |
| Процессор | 2 ядра | — |

**Операционная система:** Windows 10/11, macOS 12+, Ubuntu 20.04+, Debian 11+

> Скачать Docker Desktop: https://www.docker.com/products/docker-desktop/  
> После установки убедитесь, что Docker запущен (иконка в трее / статус `docker info`).

---

## 2. Получение исходных файлов

Для развёртывания необходимы два каталога:

```
kars-avia-backend-graphql/   ← бэкенд (содержит docker-compose.yml)
kars-avia/                   ← фронтенд
```

Оба каталога должны находиться **рядом** в одной папке, например:

```
C:\KarsAvia\
├── kars-avia-backend-graphql\
└── kars-avia\
```

## 3. Пошаговое развёртывание

### Шаг 1. Откройте терминал

- **Windows:** нажмите `Win + R`, введите `cmd` или `powershell`, нажмите Enter
- **macOS:** откройте Терминал (`Cmd + Пробел` → «Терминал»)
- **Linux:** откройте любой терминал

### Шаг 2. Перейдите в каталог бэкенда

```bash
cd путь/до/kars-avia-backend-graphql
```

Пример для Windows:
```
cd C:\KarsAvia\kars-avia-backend-graphql
```

### Шаг 3. Запустите систему

```bash
docker compose up --build
```

Эта команда автоматически:
1. Скачивает образ MongoDB 7.0 (~700 МБ, только при первом запуске)
2. Собирает образ бэкенда (устанавливает зависимости Node.js)
3. Собирает образ фронтенда (компилирует React-приложение, устанавливает Nginx)
4. Инициализирует базу данных MongoDB (Replica Set)
5. Применяет схему базы данных (Prisma db push)
6. Запускает все три сервиса

**Первый запуск занимает 5–15 минут** (зависит от скорости интернета и компьютера).  
При последующих запусках: 1–2 минуты.

Дождитесь появления сообщения:
```
karsavia-backend  | Server running on http://localhost:4000/graphql
```

## 4. Проверка работоспособности

После успешного запуска откройте браузер:

| Сервис | Адрес | Описание |
|--------|-------|----------|
| **Веб-приложение (фронтенд)** | http://localhost:3000 | Основной интерфейс |
| **GraphQL API (бэкенд)** | http://localhost:4000/graphql | API-сандбокс Apollo |
| **Проверка состояния сервера** | http://localhost:4000/health | Должен вернуть `{"status":"ok"}` |

### Вход в систему

На странице http://localhost:3000 введите тестовые учётные данные:

| Роль | Логин | Пароль |
|------|-------|--------|
| Суперадминистратор | `admin` | `admin123` |

> **Примечание:** Если учётная запись не создана, обратитесь к техническому специалисту (см. раздел 7). При первом развёртывании база данных пуста — потребуется создать администратора через GraphQL API или скрипт инициализации.

## 5. Структура сервисов

После запуска работают три контейнера Docker:

```
┌──────────────────────────────────────────────────────────┐
│                    Docker Compose Stack                   │
│                                                          │
│  ┌─────────────────┐    ┌──────────────────────────┐    │
│  │  karsavia-front │    │    karsavia-backend       │    │
│  │  (Nginx + React)│    │    (Node.js + GraphQL)    │    │
│  │  порт: 3000     │───▶│    порт: 4000             │    │
│  └─────────────────┘    └────────────┬─────────────┘    │
│                                       │                  │
│                          ┌────────────▼─────────────┐   │
│                          │    karsavia-mongo         │   │
│                          │    (MongoDB 7.0)          │   │
│                          │    порт: не проброшен     │   │
│                          └──────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

### Описание сервисов

| Сервис | Образ | Роль |
|--------|-------|------|
| `karsavia-frontend` | Nginx 1.27 + собранный React | Отдаёт статику веб-приложения |
| `karsavia-backend` | Node.js 20 Alpine | GraphQL API, бизнес-логика, WebSocket |
| `karsavia-mongo` | MongoDB 7.0 | Хранение всех данных (ReplicaSet rs0) |

### Где хранятся файлы

| Директория | Содержимое |
|-----------|------------|
| `kars-avia-backend-graphql/uploads/` | Загруженные файлы пользователей |
| `kars-avia-backend-graphql/reports/` | Сгенерированные отчёты |
| `kars-avia-backend-graphql/reserve_files/` | Файлы бронирований |
| `kars-avia-backend-graphql/logs/` | Журналы работы сервера |
| Docker volume `karsavia-mongo-data` | Данные MongoDB |

## 6. Управление системой

### Остановить систему (данные сохранятся)
```bash
docker compose down
```

### Запустить повторно (без пересборки)
```bash
docker compose up
```

### Просмотр журналов
```bash
# Все сервисы
docker compose logs -f

# Только бэкенд
docker compose logs -f backend

# Только MongoDB
docker compose logs -f mongo
```

### Полная очистка (включая данные БД)
```bash
docker compose down -v
```
> ⚠️ Эта команда удаляет все данные из базы данных. Используйте с осторожностью.

### Проверить статус контейнеров
```bash
docker compose ps
```

## 8. Дополнительная информация

### Переменные окружения

Конфигурация Docker-развёртывания находится в файле `.env.docker`.  
Шаблон для продакшн-конфигурации — `.env.example`.

Основные параметры:

| Переменная | По умолчанию | Описание |
|-----------|-------------|----------|
| `DATABASE_URL` | `mongodb://mongo:27017/avia-db?...` | Строка подключения к MongoDB |
| `JWT_SECRET` | `docker-demo-secret-...` | Секрет для подписи токенов |
| `NODE_ENV` | `dev` | Режим запуска |
| `EMAIL_ENABLED` | `false` | Включение email-уведомлений |

### Архитектура приложения

```
Браузер пользователя
        │
        │ HTTP :3000
        ▼
┌───────────────┐     HTTP/WS :4000     ┌──────────────────┐
│  React SPA    │ ──────────────────▶  │  GraphQL API     │
│  (Nginx)      │                       │  (Apollo Server) │
└───────────────┘                       │  Node.js 20      │
                                        └────────┬─────────┘
                                                 │ Prisma ORM
                                                 ▼
                                        ┌──────────────────┐
                                        │  MongoDB 7.0     │
                                        │  ReplicaSet rs0  │
                                        └──────────────────┘
```

**Технологии:**
- **Фронтенд:** React 18, Vite, Apollo Client, Material UI
- **Бэкенд:** Node.js 20, Express, Apollo Server 4 (GraphQL), Prisma ORM
- **База данных:** MongoDB 7.0 (ReplicaSet)
- **Контейнеризация:** Docker, Docker Compose

### Порты

| Порт | Сервис | Протокол |
|------|--------|----------|
| 3000 | Фронтенд (веб-интерфейс) | HTTP |
| 4000 | Бэкенд (GraphQL API + WebSocket) | HTTP / WS |

MongoDB работает **только внутри Docker-сети** и не доступна снаружи.
